# Frontend Bundle

## Use When

- The task changes user-facing UI, routing, forms, dashboards, visual design, responsive layout, or browser interaction.

## Skills

- `frontend-ui-engineering`
- `test-driven-development`
- `browser-ui-verification`
- `visual-regression-and-layout-qa`

## References

- `references/accessibility-checklist.md` when accessibility risk exists

## Completion Requires

- Relevant tests, lint, typecheck, or a stated reason they could not run
- Real browser journey verified through visible UI operations
- Console checked
- Screenshots captured for relevant states
- Layout audit or manual layout QA across relevant viewports
- Remaining risks named

## Failure Conditions

- No browser evidence
- Mobile or narrow viewport untested for responsive UI
- Console errors unexplained
- Known overlap, overflow, clipping, or hidden controls remain unresolved
