# Review Bundle

## Use When

- Reviewing code written by an agent, human, or prior session before accepting or merging it.

## Skills

- `code-review-and-quality`
- `test-driven-development`
- `security-and-hardening` when user input, auth, or secrets are involved
- `performance-optimization` when runtime performance can regress
- `browser-ui-verification` for user-visible changes

## Completion Requires

- Findings first, ordered by severity
- Tests and evidence reviewed
- UI evidence checked when applicable
- Security and reliability risks named
- Remaining gaps stated clearly

## Failure Conditions

- Review only says "looks good"
- Tests are listed but not interpreted
- UI changes lack screenshots or browser proof
- Security-sensitive changes are reviewed only for style
