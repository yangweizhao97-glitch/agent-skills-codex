# Ship Bundle

## Use When

- Preparing a meaningful change for release, sharing, or production use.

## Skills

- `shipping-and-launch`
- `ci-cd-and-automation`
- `documentation-and-adrs`
- `observability-and-instrumentation`
- `git-workflow-and-versioning`

## Completion Requires

- Validation checks pass
- Release notes or changelog updated
- Rollback or recovery path named when relevant
- Monitoring or evidence expectations documented
- Version tag planned or created

## Failure Conditions

- No release notes
- No validation run
- No known rollback path for risky changes
- Documentation says the feature is complete without evidence
